API Documentation

Based on simple REST principles, our Web API enpoints return JSON metadata about smart phone specifications.

The base address of this API is http://localhost:3000/. The API provides a set of endpoints, each with its own unique path.

REQUESTS

This API is based on REST principles, thus, data resources are accessed via standard HTTPS requests in UTF-8 format to an API endpoint.

Where possible, this API uses appropriate HTTP verbs for each action:

METHOD ACTION
GET Retrieves resources from the database
POST Adds additional resources to the database
PUT Updates resources in the database
DELETE Deletes resources in the database

ENDPOINTS

The following endpoints will determine the actions of the Web API:

PATH DESCRIPTION HTTP METHOD CRUD FUNCTION
http://localhost:3000/phones Retrieves all phones from the database including all specifications such as brand, model, image, os, and screensize. GET - safe and idempotent Retrieve
http://localhost:3000/phones/:id Retrieves specific phone from the database based on a given id. GET - safe and idempotent Retrieve
http://localhost:3000/add-phone Adds new phone to database with specifications such as brand, model, image, os, and screensize. POST - not idempotent Create
http://localhost:3000/remove-phone Removes all phones from the database. DELETE - idempotent but not safe Delete
http://localhost:3000/remove-phone/:id Removes a phone from the database with a specific id that it is referring to. DELETE - idempotent but not safe Delete
http://localhost:3000/update/:id Updates a phone from the database with a specific id that it is referring to. PUT - idempotent but not safe Update
